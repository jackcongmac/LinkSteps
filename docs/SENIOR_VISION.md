# LinkSteps Senior: Product Vision & Strategy

> **最后更新**：2026-03-31（v0.4 — Phase 1 MVP 核心功能已交付）

## 1. 核心理念 (Core Philosophy)
**"关怀，不是监管" (Care, not Surveillance).** LinkSteps Senior 致力于解决 30-45 岁独生子女一代对 70 岁以上长辈的"远距离关怀焦虑"。我们不追求 ICU 级别的医疗数据实时监控，而是通过提取基础行为数据和环境因子，为晚辈提供"确定性"，为长辈提供"零负担的陪伴"。

## 2. 核心战略 (Strategic Pillars)

### A. 双端非对称设计 (Asymmetric UX)
* **长辈端 (Parent UI)**：去标签化、极简主义。核心是一个巨大的绿色【平安扣】按钮和语音接收器。绝不出现复杂的医疗图表或令老人焦虑的"警告"。
* **晚辈端 (Carer UI)**：全知看板。聚合所有数据流，利用 AI 翻译成具体的行动建议（如："妈妈今早步数较少，建议打个视频"），降低晚辈的认知负荷。

### B. 零门槛设备接入 (Zero-Struggle Onboarding)
* **原则**：晚辈干脏活，长辈点同意。
* **路径**：彻底放弃让老人操作"蓝牙配对"。全面采用"云端直连"或"微信生态授权"。晚辈发送链接，长辈一键授权，后台静默拉取数据。

### C. 数据的优雅降级 (Graceful Degradation of Data)
AI 引擎必须适应不同老人的佩戴习惯，提供分层的关怀价值：
* **有高级设备 (High Data)**：基于 HRV、深度睡眠、步数计算基线偏移，提供精准的疲劳和健康预警。
* **只有基础设备/手机 (Sparse Data)**：依赖气象数据（气压/温差）、首次亮屏时间、步数，提供"生活节律"与"环境关怀"提醒。
* **完全无数据 (No Data)**：依靠【平安扣】的主动 Check-in 和 TTS 语音留言，维持轻量级的情感连接。

## 3. 演进路线图 (Phased Roadmap)

* **Phase 1: 基础体感期 (MVP)** ✅ 已完成
  * **输入**：健康模拟器（步数 + 心率） + 第三方气象 API + 手动平安扣
  * **输出**：
    - 双向文字/语音消息；长辈快捷请求（回微信/电话）
    - 实时 FamilyTimeline（平安信号 + 消息 + 系统警报）
    - 心率异常自动检测与 Timeline 记录
    - 长辈档案（姓名 + 年龄 + 关系 + 性别）可编辑
    - 7 天数据保留（pg_cron）
    - 长辈端接收消息 + 已读功能

* **Phase 2: 睡眠感知期** 🔄 部分实现（dev 模拟器）
  * **输入**：接入基础手环的睡眠时长数据
  * **已实现**：sleep_sessions 表 + SleepInsightsCard（dev 模拟）+ 夜间 awake 告警
  * **待实现**：真实 OAuth 设备接入（华为/小米）；晚辈邀请二维码；晚辈个人档案
  * **输出**：清晨睡眠简报，预测长辈当天精力状态

* **Phase 3: 生理预警期** 📋 待规划
  * **输入**：接入 HRV、连续血氧等高阶指标
  * **输出**：结合基线偏移，提供心血管或隐性疾病的预防性干预建议
  * **功能**：SOS 全屏弹窗；多晚辈协同；历史趋势图；fall_detected 全面集成

## 4. 黄金军规 (The Golden Rules)
1. **永远不报错给老人**：设备断连或数据异常，只提醒晚辈，不惊扰长辈。
2. **不说黑话**：AI 建议中禁止使用晦涩的医学缩写，将"HRV 低"翻译为"似乎有些疲劳"。
3. **制造连接契机**：每一次数据的更新，都是一次"不需要找话题"的互动理由。
4. **Fire-and-forget 原则**：长辈端操作（平安扣、发语音）UI 始终显示成功，失败静默 log，不打扰长辈。
5. **所有活动留痕**：晚辈端触发的任何告警（心率异常等）自动写入 FamilyTimeline，形成完整关怀记录。
